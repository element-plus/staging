Tema personalizado Obsoleto desde 1.1.0-beta.1 #

Element Plus utiliza la metodología BEM en CSS con la finalidad de que puedas sobrescribir los estilos fácilmente. Pero si necesita reemplazar estilos a gran escala, p.ej. cambiar el color del tema de azul a naranja o verde, tal vez sobreescribirlos uno por uno no es una buena idea. proporcionamos cuatro maneras de cambiar las variables de estilo.

Cambiando el color del tema #

Si solo quieres cambiar el color del tema del Elemento Plus, se recomienda sitio web de vista previa del tema . Element Plus utiliza un color azul brillante y amigable como tema principal. Al cambiarlo, puede hacer que Element Plus esté más conectado visualmente a proyectos específicos.

El sitio web anterior le permite previsualizar el tema de un nuevo color de tema en tiempo real y puede generar un paquete de estilo completo basado en el nuevo color del tema para que se descargue directamente (para importar nuevos archivos de estilo en tu proyecto, por favor refiérase a el 'Importar tema personalizado' o 'Importar tema de componente a petición' parte de esta sección).

Actualizando variables SCSS en tu proyecto #

theme-chalk esta escrito en SCSS. Si su proyecto también utiliza SCSS, puede cambiar directamente variables de estilo Element Plus. Para ello, solo necesita crear un nuevo archivo de estilos, por ejemplo, element-variables.scss:

/* theme color */
$--color-primary: teal; /* icon font path, required */
$--font-path: '~element-plus/lib/theme-chalk/fonts';
@import '~element-plus/packages/theme-chalk/src/index';

Luego, en el archivo de entrada de tu proyecto, importa este archivo de estilo en lugar del CSS construido por Element:

import Vue from 'vue'
import ElementPlus from 'element-plus'
import './element-variables.scss'
import App from './App.vue';

const app = createApp(App)
app.use(ElementPlus)

TIP

Tenga en cuenta que es necesario anular la ruta de la fuente del icono a la ruta relativa de los archivos de fuente del elemento .

CLI para generar temas #

Si tu proyecto no utiliza SCSS, puedes personalizar temas con nuestra herramienta de tema CLI:

Instalación #

Primero instale el generador de temas de forma global o local. La instalación local es recomendada porque de esta manera, cuando otros clonan tu proyecto, npm lo instalará automáticamente para ellos.

npm i element-theme -g

Luego instale el tema chalk desde npm o GitHub.

# desde npm
npm i element-theme-chalk -D

# desde GitHub
npm i https://github.com/ElementUI/theme-chalk -D

Inicializar archivo de variables #

After successfully installing the above packages, a command named et is available in CLI (if the packages are installed locally, use node_modules/.bin/et instead). Ejecute -i para inicializar el archivo variable que saldrá a element-variables.scss por defecto. Y puede especificar su directorio de salida como quiera.

et -i [custom output file]

> ✅ Generator variables file

En element-variables.scss puedes encontrar todas las variables que usamos para estilizar Element Plus y que están definidas en formato SCSS. Aquí hay un snippet:

$--color-primary: #409EFF !default;
$--color-primary-light-1: mix($--color-white, $--color-primary, 10%) ! efault; /* 53a8ff */
$--color-primary-light-2: mix($--color-white, $--color-primary, 20%) !default; /* 66b1ff */
$--color-primary-light-3: mix($--color-white, $--color-primary, 30%) ! efault; /* 79bbff */
$--color-primary-light-4: mix($--color-white, $--color-primary, 40%) !default; /* 8cc5ff */
$--color-primary-light-5: mix($--color-white, $--color-primary, 50%) ! efault; /* a0cfff */
$--color-primary-light-6: mix($--color-white, $--color-primary, 60%) ! efault; /* b3d8ff */
$--color-primary-light-7: mix($--color-white, $--color-primary, 70%) !default; /* c6e2ff */
$--color-primary-light-8: mix($--color-white, $--color-primary, 80%) ! efault; /* d9ecff */
$--color-primary-light-9: mix($--color-white, $--color-primary, 90%) ! efault; /* ecf5ff */

$--color-success: #67c23a !default;
$--color-warning: #e6a23c !default;
$--color-danger: #f56c6c !default;
$--color-info: #909399 !default;

...

Modificando variables #

Solo edita element-variables.scsssssss, por ejemplo, cambiando el color del tema a rojo:

$--color-primary: red;

Construyendo el tema #

Después de guardar el archivo de variables, usa et para construir tu tema. Puede activar el modo de reloj `` añadiendo un parámetro -w. Y si personalizó la salida del archivo de la variable , necesita añadir un parámetro -c y el nombre del archivo variable. Por defecto el archivo de tema de construcción se coloca dentro de ./theme. Puede especificar su directorio de salida con el parámetro -o.

et

> ✅ build theme font
> ✅ build element theme

Usar tema personalizado #

Cubriremos cómo utilizar tu tema personalizado en tu proyecto en esta sección

Importar un tema personalizado #

Importar tu propio tema es igual que importar el tema por defecto, solo esta vez que importas el archivo construido a partir de "Roller de Temas en línea" o "Herramienta CLI":

import { createApp } from 'vue'
import '../theme/index.css'
import ElementPlus from 'element-plus'

createApp(App).use(ElementPlus)

Importar un tema de componente bajo demanda #

Si está utilizando babel-plugin-component para importación bajo demanda, sólo modifique . abelrc y especifica styleLibraryName a la ruta donde tu tema personalizado se encuentra relativo a . abelrc. Ten en cuenta que ~ es requerido:

{
  "plugins": [
    [
      "component",
      {
        "libraryName": "element-plus",
        "styleLibraryName": "~theme"
      }
    ]
  ]
}

Si no está familiarizado con babel-plugin-component, consulte Quick Start. Para obtener más detalles, consulte el repositorio de proyecto de element-theme.
