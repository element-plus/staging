Select V2 virtualized selectorbeta
#

TIP

Este componente todavía está en pruebas, si encuentra algún error o problema, por favor infórmenos en GitHub para que lo arreglemos.

Fondo #

En algunos casos de uso, un selector puede terminar cargando decenas de miles de filas de datos. Renderizar esa cantidad de datos en el DOM podría ser una carga para el navegador, lo que puede resultar en problemas de rendimiento. Para una mejor experiencia de usuario y desarrollador, decidimos añadir este componente.

Uso Básico #

El selector más simple

Selección múltiple #

El selector básico de selección múltiple con etiquetas

Ocultar etiquetas extra cuando los elementos seleccionados son demasiados. #

Se pueden contraer a un texto utilizando el atributo collapse-tags. Puede comprobarlo cuando el ratón pasa el cursor sobre el texto usando el atributo collapse-tags-tooltip.

Selección múltiple filtrable #

Cuando hay demasiadas opciones, puede usar la opción filterable para habilitar la función de filtro y así encontrar la opción deseada

Deshabilitar el selector y las opciones de selección #

Puede elegir desactivar el selector en sí mismo o con option.

Grupo de opciones #

Puede agrupar la opción como quiera, siempre y cuando los datos satisfagan el patrón.

Personalizar las opciones de renderizado #

Se puede definir una plantilla propia para renderizar la opción en la ventana emergente.

Limpiando el selector #

Se pueden borrar todas las opciones seleccionadas a la vez, también aplicable para una única selección.

Crear nuevos ítems #

Crear y seleccionar nuevos elementos que no están incluidos en las opciones de selección

Al usar el atributo allow-create, los usuarios pueden crear nuevos elementos escribiendo en el input. Tenga en cuenta que para que allow-create funcione, filterable debe ser true.

TIP

Es mejor establecer :reserve-keyword="false" cuando se use allow-create
